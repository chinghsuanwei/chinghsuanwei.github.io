使用Node-js 網站自動登入程式

最近因為有上某個實體課程 但它的課程訂課方式是 每天到了某個時間點 開放大家去搶課

每天當搶課時間到的時候 光是refresh課程列表 就卡了好幾秒鐘

看到列表時 大部分的好課程都全部都顯示已滿

對此我感到很不滿 怎麼訂課是這種形式 錢都繳了 但卻都上不到課

後來就決定寫一個自動訂課的程式

我想各位也可能會遇到相似的問題 例如: “需要在某個時間點 登入某個網站 trigger 某個按鈕之類的”

若是如此 那可以參考本篇一下

以下是記錄我遇到的困難, 解法 和 可以改進的地方

為了避免提到是哪個系統 我們以Evernote官方網站當作範例

我們會使用到的工具有 postman 和 google chrome

首先我們必須要拿到登入的request

打開google chrome 來到登入頁面

按下 F12 會打開 Debug tool

輸入完帳密後 嘗試擷取network封包

這裡會遇到的問題可能是refresh的時候 chrome Network的頁面也會刷新 這裡只要將preserve log的option 打勾

Google Chrome 就不會在更新頁面的時候 將request log清除

之後按下Login之後 我們就可以抓到 chrome送給 evernote server的 login request了

在來我們需要將request import到postman

對request按下右鍵 選取 Copy as cURL(bash)

再來將內容import至postman

注意: 這裡要使用Windows版的postman 不要使用google chrome 外掛版本 import會失敗

此時我們就將request匯入postman 再來就是寫一個Node-js程式 讓它可以自動登入 和 自動訂課

Postman有一個很強大的功能 就是可以把request本身 轉換成各種常用的程式語言 像是Java, Objective-C, Javascript…… 等等

因為我們要用Node-js 所以這裡選擇 Javascript - Fetch

所以你會得到類似下面的code

var myHeaders = new Headers();
myHeaders.append(“authority”, “www.evernote.com");
myHeaders.append(“cache-control”, “max-age=0”);
myHeaders.append(“origin”, “https://www.evernote.com");
myHeaders.append(“upgrade-insecure-requests”, “1”);
myHeaders.append(“content-type”, “application/x-www-form-urlencoded”);
myHeaders.append(“user-agent”, “Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/80.0.3987.163 Safari/537.36”);
myHeaders.append(“sec-fetch-dest”, “document”);
myHeaders.append(“accept”, “text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,/;q=0.8,application/signed-exchange;v=b3;q=0.9”);
…

var urlencoded = new URLSearchParams();
…

var requestOptions = {
method: ‘POST’,
headers: myHeaders,
body: urlencoded,
redirect: ‘follow’
};

fetch(“https://www.evernote.com/Login.action", requestOptions)
.then(response => response.text())
.then(result => console.log(result))
.catch(error => console.log(‘error’, error));

你可以生成一個純文字文件將上面程式碼貼上去

var fetch = require(‘node-fetch’);
const {Headers} = require(‘node-fetch’);
const {URLSearchParams} = require(‘url’);

之後在前面貼上

照理來說就能正確地執行

記得用npm下載相依的Library

目前遇到的問題是 在 myHeaders.append(“cookie”, …) 裡面會有一個ㄧsession-id

使用的時候 還是要手動登入一次 去查cookie中的session id為何 並修改程式 或 將其帶入程式

我在猜應該是我同時在使用網頁的關係 導致session-id也一直更新 or 伺服器本來就會刷新seesion-id 這部分我目前還沒想到方法 但對我來說 我覺得已經足夠

Rakuten Codility online programming test

Only one question

Q1.
There has 3 rules for string

  1. a string must start with a consonant
  2. no two consecutive vowels
  3. no two consecutive consonants

just only consider upper-case

At first time, I just wanted to use bruteforce-way to solve the problem. But I thought I could not pass the large test if I used bruteforce-way.
Then I spend like 20-min to figure out the question.

It’s just a permutations problem.

The count of vowels must equal or less than 1 to consonants

So for alphabets “AEBBC”, there has 2! * (3! / 2!) = 6 kind of permutations

  1. “BABEC”
  2. “BACEB”
  3. “CABEB”
  4. “CEBAB”
  5. “BECAB”
  6. “BEBAC”

Codility

Since I am not very sure that all paricipate will get same question, so I decide to publish these in Aug.

In June late, I take participation from
【BizReach Hackathon in Tokyo 2019】Invitation to qualifying round coding challenge.

There has 2 questions. You cannot use C and C++. Fortunately, there still has Java option.

The first one is easy.

Q1.
A multiply by B equal C.
Calculate the number of 1’s in C.

The range of A and B is 1,000,000,000.

So just to notice to use long long primitive type to multiply to avoid overflow.

Q2.
This one is a little tough.
The description of scenario is so long, but the real question is to find a index in array which can guarantee the numbers at right to the target index(included) are larger than left ones.

For example,
[1, 2, -1, 4, 5]

Then the anwser is 3. Since 4 and 5 are larger than 1, 2, and -1.

p.s There must exist an index can seperate two group.

I didn’t find O(n) solution in contest.
But later, I think a while the pseudo code for O(n) should be.

size of array is N

int max_idx = N-1;
int max = array[N-1];

//Max won’t appear in first element
//We need to find leftmost max
for(int i=N-2; i>0; i++)
{
if(array[i] >= max)
{
max = array[i];
max_idx = i;
}
}

int rmin = array[max_idx];

//find the smallest number at right size
for(int i=max_idx+1; i<=N; i++)
{
if(array[i] < rmin)
{
rmin = array[i];
}
}

int target_idx = 0;
int lmax = array[0];

for(int i=1; i<max_idx; i++)
{
if(array[i-1] > array[i])
{
target_idx = -1;
}
else if(array[i] > array[i-1])
{
//target idx must bigger than lmax and rmin, at same time target_idx is not assigned
if(array[i] > lmax && array[i] > rmin && target_idx == -1)
{
target_idx = i;
}

//keep track lmax
if(array[i] > lmax)
{
    lmax = array[i];
}

}
}

return target_idx == -1? max_idx : target_idx;
