Codility

Since I am not very sure that all paricipate will get same question, so I decide to publish these in Aug.

In June late, I take participation from
【BizReach Hackathon in Tokyo 2019】Invitation to qualifying round coding challenge.

There has 2 questions. You cannot use C and C++. Fortunately, there still has Java option.

The first one is easy.

Q1.
A multiply by B equal C.
Calculate the number of 1’s in C.

The range of A and B is 1,000,000,000.

So just to notice to use long long primitive type to multiply to avoid overflow.

Q2.
This one is a little tough.
The description of scenario is so long, but the real question is to find a index in array whic can guarantee the numbers at right to the target index(included) are larger than left ones.

For example,
[1, 2, -1, 4, 5]

Then the anwser is 3. Since 4 and 5 are larger than 1, 2, and -1.

p.s There must exist an index can seperate two group.

I didn’t find O(n) solution in contest.
But later, I think a while the pseudo code for O(n) should be.

size of array is N

int max_idx = N-1;
int max = array[N-1];

//Max won’t appear in first element
//We need to find leftmost max
for(int i=N-2; i>0; i++)
{
if(array[i] >= max)
{
max = array[i];
max_idx = i;
}
}

int rmin = array[max_idx];

//find the smallest number at right size
for(int i=max_idx+1; i<=N; i++)
{
if(array[i] < rmin)
{
rmin = array[i];
}
}

int target_idx = 0;
int lmax = array[0];

for(int i=1; i<max_idx; i++)
{
if(array[i-1] > array[i])
{
target_idx = -1;
}
else if(array[i] > array[i-1])
{
//target idx must bigger than lmax and rmin, at same time target_idx is not assigned
if(array[i] > lmax && array[i] > rmin && target_idx == -1)
{
target_idx = i;
}

//keep track lmax
if(array[i] > lmax)
{
    lmax = array[i];
}

}
}

return target_idx == -1? max_idx : target_idx;

Chinese is fucking hard

最近在學語言才開始注意中文也有一些很難的規則

例如:
一枝筆
一幅畫
一顆球

要講得像母語人士 必須要學會量詞 如果都用”一個”的話 也能表達意思 但聽起來很怪
(X) “我要一個水”
(O) “我要一杯水”

我最近發現 有一個例子 自己都覺得好像蠻難的 “一副”要打架的樣子 可能很難理解為何這裡要用一副
通常都是”一副眼鏡”之類的
其實我也不知道…

再來是發音的部分
台灣還是主要用注音符號系統 但對於國際化來說 注音對外國人來講太過困難 要快速獲得習得語言的成就感的方法
還是使用”漢語拼音” 我想大部分的外籍人士都會選擇漢語拼音

之前看過一個外國人學中文 舉一個 “why chinese is hard?” 的例子

溫(wēn) means “warm”
聞(wén) means “smell”
吻(wěn) means “kiss”
問(wèn) means “ask”

對外國人來說 一個wen 有四個聲音 還不止 搞得夠複雜了 中國還一大堆漢字
不過我餐廳小姐看到我在練習寫韓文 他就和我聊天 她就說 “我覺得中國字很美”
一想想 也對 中國字 有一部分是象形字 而且中文有書法 就是一種書寫中文的藝術作品
這應該是其他國家沒有的 就因為中文字複雜 還有很多字都有他的故事 所以也讓很多西方人 喜歡刺中文字在身上

之前有看台灣大學的 open course “英語語音學”
http://ocw.aca.ntu.edu.tw/ntu-ocw/index.php/ocw/cou/101S102/1
教授 Karen Steffen Chung 史嘉琳 他發音已經算很標準了
但還是能隱約聽到一些口音 就連語音學教授都有口音了 其他很多大人要去學基本上很難避免
語言還是從小開始學是最輕鬆的
我覺得我學長講一句話我還算蠻認同的 其實應該要慶幸自己誕生在 中文語系國家 比起學英文 學中文要困難許多
但英文還是Global language 因為文件, 電腦, … 等等的東西 都已經被英美制定好了 注定讓英語變成所謂的世界共通語
所以英文還是一定要會的
但我覺得學語言還是蠻開心的 像雙關語的笑話 一定要懂該國家的語言才能聽懂

最後分享一首歌 生僻字
裡面使用很多很難的中文字 和 我們平常根本不會用到的字 和 成語
但聽完也讓我感覺中文字真偉大

Macbook Pro 2010 mid

If you are using Macbook pro 2010 mid, remember to install gfxCardStatus and turn on 僅Integrated晶片. Otherwise your computer will crash often when OS decide to use external graphic card.
Reference: https://discussions.apple.com/thread/4766577?start=4695&tstart=0

Notice! After upgrade to 10.12, the gfxCardStatus 2.3 version won’t work. I need to use previous version like v1.7.5, to force turning on Internal Graphic Card only mode.
