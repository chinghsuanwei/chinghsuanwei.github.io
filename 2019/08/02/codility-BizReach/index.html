Codility

Since I am not very sure that all paricipate will get same question, so I decide to publish these in Aug.

In June late, I take participation from
【BizReach Hackathon in Tokyo 2019】Invitation to qualifying round coding challenge.

There has 2 questions. You cannot use C and C++. Fortunately, there still has Java option.

The first one is easy.

Q1.
A multiply by B equal C.
Calculate the number of 1’s in C.

The range of A and B is 1,000,000,000.

So just to notice to use long long primitive type to multiply to avoid overflow.

Q2.
This one is a little tough.
The description of scenario is so long, but the real question is to find a index in array which can guarantee the numbers at right to the target index(included) are larger than left ones.

For example,
[1, 2, -1, 4, 5]

Then the anwser is 3. Since 4 and 5 are larger than 1, 2, and -1.

p.s There must exist an index can seperate two group.

I didn’t find O(n) solution in contest.
But later, I think a while the pseudo code for O(n) should be.
